SANs (Storage Area Network)

é uma rede especializada que conecta servidores a dispositivos de armazenamento de forma dedicada e de alto desempenho.

Camada 1 - Fisisca

unidade: Bits

ela é a responsavel por transmitir os 0/1 entre os dispositivos atraves do meio fisico

nela que existe as topologias de rede.

define caracteristicas FISICAS dos meios de transmissão

Ela não se importa com nada apenas transmite a sequencia de numeros adiante.

unidade de dados são bits.

Meios de transmissão

- Cabos UTP (par trançado sem blindagem)

- Cabos coaxial (o grosso)

- Cabos de fibra opitica (transmissão por luz)

Sinais

- Eletricos, opiticos, radio (wireless)

Equipamentos da camada Fisica

- Repetidores

Que nem o minecraft.

- Hubs

Ao receber uma mensagem compartilha para todos os dispositivos conectados.

Grita para todo mundo na esperança do ouvinte ouvir.

- Modem fisico

Converte sinal analogico em digital e vice versa.

Usado na conexão a internet, geralmente integrado ao roteador.

Camada 2 - ENLACE

unidade: quadros(frames)

Essa camada serve para detectar erros que ocorrem na camada fisica.

ela é um intermediario entre a camda Fisica e a de Rede.

Sua unidade de dados são quadros(frames) que são os conjuntos de bits.

Quadros

[ Cabeçalho ] [ Dados úteis (Payload) ] [ FCS ]

Cabeçalho

Como uma etiqueta ele traz metadados como:

Endereço de origem: Mac do remetente

Endereço de destino: Mac do receptor

Tipo de protocolo: usado na camada superiro tipo IPv4 ou IPv6

Controle de sequencia: caso a transmissão precise ser ordenada

FCS

É a sequencia de verificação inserida no final do quadro

Serve para detectar erros de transmissão referentes a camada fisica.

O receptor (quem ta recebendo a msg) refaz o calculo e verifica se bate com o FCS

O CALCULO É FEITO COM Cyclic Redundancy Check (CRC)divisão binaria e o resto vira FCS.

Se o calculo não coincidir o quadro é descartado ou solicirtado novamente.

A camada de ENLACE se divide em duas SUB-camadas

LLC Logical Link Control

- interface logica com a camada da rede

- identifica o protocolo tipo IPv4, IPv6

- fala para o MAC qual protocolo esta sendo usado na camada 3

- pode ajudar no controle de fluxo. nem sempre. nem é sua função. so pode. tranmissor não sobrecarregar o receptor com dados.

MAC Media Access Control

- Define como os dispositivos acessam o meio Endereço MAC

- Monta quadros, adiciona o Cabeçalho calcula e inseri o FCS

- A camada basicamente direciona o "mac unico" de todos os dispositivos na rede

Equipamentos da camada de ENLACE

Switches

- usam o Endereço mac para mandar a mensagem diretamente ao destinatario

- é inteligente, não grita a msg pra todo mundo apenas pro destino.

- o Hub burro que faz isso. (hub não é dessa camada)

Placa de rede

- conecta o dispostivo a rede. (todos os destinos TEM)

- traduz dados digitais do computador em sinais eletricos opticos ou de radio

- possui um endereço mac unico, usado na identificação fisica na rede.

- tipos de placa: Ethernet(cabo), Wi-Fi(sem fio), USB, PCI/PCIe

VLANs (n é equipamento pq é logico n é real)

- São configuradas por Switches. basicamente "redes lógicas"(n fisicas)

- isolam grupos de dispositivos como se estivessem em redes diferentes

- Ela utiliza o endereço MAC dos dispositivos e adiciona uma tag VLAN no quadro Ethernet (não no pacote IP)

Bridges (Pontes de rede)

- interliga dois segmentos de rede, permitindo que funcionem como uma rede so.

- Geralmente conectada entre Switches.

- Aprende os caminhos MAC disponiveis e se tiver manda a msg para la (geralmente para um switche)

Camada 3 - REDE

unidade: pacotes(Packets)

Ela cuida do endereçamento lógico, roteamento, e garante que os dados cheguem ao destino, mesmo que estejam em redes diferentes.

- encaminha pacotes entre dispositivos em redes diferentes

- escolhe a melhor rota para os dados

- realiza o endereçamento lógico (IP)

- gerencia o congenstionamento, a fragmentação e o controle de tráfego

- enquanto a camada de ENLANCE trabalha com quadros(frames)a camada de rede organiza os dados em PACOTES

PACOTEs

Endereço IP de origem / Endereço IP de destino / informações de controle (tempo de vida, protocolo, etc)

Endereços IP

- Essa camada utiliza endereços IP em vez de MAC

- Permite que dispositivos se comuniquem entre redes distintas

- IPv4 -> 192.168.0.1 IPv6 -> 2001:db8::1

- enquanto o endereço mac é o "numero da porta da casa" o IP é o "CEP DA CIDADE" mais abrangente e roteável.

Roteamento

- escolhe o melhor caminho entre origem e destino, feito por dispositivos chamados roteadores

- Roteamento estatico: feito manualmente pelo administrador.

- Roteamento dinamico: baseado em protocolos que escolhem a rota automaticamente

- Com os protocolos é melhor pois a rede pode mudar o caminho se tiver erros.

Protocolos de roteamento

(INFELIZMENTE É IMPORTANTE CAIU NA QUESTÃO DE ANALISTA DA FAURGS)

Protocolo Algoritmo Baseado em... Características Principais
RIP (Routing Information Protocol) Vetor de Distância Número de saltos - vizinho conta caminho Simples, até 15 saltos, convergência lenta, usa UDP
OSPF (Open Shortest Path First) Estado de Ligação Custo dos links - calcula com base no mapa individual total da rede / gps propio Rápido, usa algoritmo de Dijkstra, escalável para redes grandes
EIGRP (Cisco) Híbrido Métrica composta (largura de banda, atraso, etc.) Combina distância e estado, rápido, exclusivo da Cisco
BGP (Border Gateway Protocol) Vetor de Caminho Lista de sistemas autônomos - caminho passo a passo Usado na Internet, altamente escalável, baseado em políticas
IS-IS Estado de Ligação Custo dos links - calcula com base no mapa individual total da rede Similar ao OSPF, robusto, usado por operadoras em redes grandes

Fragmentação de pacotes

- Se o pacote é muito grande para a rede de destino ela pode ser quebrada em partes.

- cada parte chega separadamente e é agrupada no destino (host)

- tambem ocorre se for muito grande para o proximo enlace (transmissão fisica, cabo ou ponto entre dois roteadores)

- muito utilizada principalmente no protocolo IPv4, quem "quebra" são os roteadores

- MTU (Maximum Tranmission Unit) é o tamanho maximo que um quadro ou pacote pode ser transmitido sem ser fragmentado.

- cada enlace fisico tem seu propio MTU, e o roteador consulta o do seguinte para decidir se fragmenta ou não.

Protoclos comuns da camada de REDE

Protocolo Função
IP (Internet Protocol) Endereçamento e roteamento entre redes
ICMP (Internet Control Message Protocol) Diagnóstico de rede e mensagens de erro (ex: ping)
ARP (Address Resolution Protocol) Traduz IP para MAC (interface entre Camadas 2 e 3)
RARP (Reverse ARP) Traduz MAC para IP (menos usado atualmente)

Equipamentos da camada de REDE

- Roteadores

Leem o IP e decidem para onde mandar esses pacotes.

Escolhem o melhor caminho por meio de um Protocolo de Roteamento (Se for dinamico)

Tambem fazem a Fragmentação de pacotes. (so se necessario)

- Hosts

Qualquer dispositivo final que possui um endereço IP e participa da comunicação da rede

Fazem a reagrupação da Fragmentação feitas pelos roteadores.

- Firewall (Camadas Lógicas em Roteadores ou Hosts)

Um dispositivo ou software que analisa o tráfego de rede e aplica regras de segurança.

Pode estar presente tanto em roteadores (protegendo a rede inteira) quanto em hosts (protegendo o dispositivo individual).

Na camada de REDE, bloqueia IPs específicos, redes inteiras ou até protocolos (ex: bloquear tráfego IPv4).

Camada 4 - TRANSPORTE

unidade: segmentos (tcp) / datagramas (udp)

Ela garante que os dados cheguem ao destino com confiabilidade (se não for UTP), ordem e integridade.

- segmenta e remonta os dados

- estabelece conexões fim a fim entre dispositivos

- contrala o fluxo (sem sobrecarregar o receptor)

- detecta e corrigi erros (em protocolos seguros / TPC)

- multiplexa varias aplicações simultaneas pelos numeros de porta

Numeros de porta e multiplexação

- São identificadores usados pela Camada de TRANSPORTE para diferenciar quais aplicações estão se comunicando em um mesmo dispositivo.

- o IP leva os dados até o dispositivo já a porta diz qual aplicativo dentro deve receber os dados.

- já a multiplexação permite com que varias aplicações compartilhem o mesmo IP simultaneamente, cada um com sua porta.

- Com a multiplexação posso jogar enquanto envio email e vejo um site sobre como jogar. Por exemplo:
Usando email -> porta 25(SMTP)
navegando na web -> 443(HTTPS)
Jogando -> porta 27015(Steam Games)

Unidade de dados Segmento(tcp) / Datagrama(UDP)

ao receber os dados da camada superior (Aplicação), a cammada de TRANSPORTE:

- divide em segmentos menores

- adiciona informações como porta de origem/destino, numero se sequencia (ordem que foi dividido), checksum (verificação de integridade de dados no nivel Lógico, como o FCS porem na 4 camada.)

- no destino remonta os dados na ordem certa (usando o numero de sequencia)

confiabilidade e controle

Recurso Função
Controle de fluxo Evita que o receptor receba mais do que pode processar
Controle de erro Detecta falhas e solicita retransmissão (em protocolos confiáveis)
Controle de ordem Garante que os dados cheguem na sequência correta

Protocolos da camada de TRANSPORTE

Protocolo Tipo Características
TCP Confiável Usa controle de sequência, confirma recebimento, retransmite se necessário
UDP Não confiável Mais rápido, não garante entrega nem ordem. Usado em vídeos, jogos, DNS

A camada de transporte atua dentro dos hosts, não em equipamentos intermediários como roteadores.

TCP 3way handshake

Antes de enviar os dados o tcp realiza um aperto de mão de 3 etapas:

- (SYN (Synchronize)) Cliente envia um pacote SYN ao servidor com um número de sequência aleatório (ex: SEQ=100)

- (SYN-ACK (Synchronize-Acknowledge)) Servidor responde com SYN-ACK (confirma SEQ=100 e envia seu próprio SEQ=300).

- (ACK (Acknowledge)) Cliente confirma com ACK=301 (próximo número esperado).

Cada pacote enviado tem um número de sequência (SEQ) e número de confirmação (ACK).

- O receptor envia um ACK confirmando o recebimento.

- Se um ACK não chegar, o TCP reenvia o pacote.

TCP 4-Way Handshake

- FIN (Cliente avisa que vai fechar).

- ACK (Servidor confirma).

- FIN (Servidor também fecha).

- ACK (Cliente confirma).

estrutura do cabeçalho TCP

Campo Tamanho Descrição
Porta de Origem 16 bits Porta do remetente (ex: 5000)
Porta de Destino 16 bits Porta do destinatário (ex: 80 para HTTP)
Número de Sequência (SEQ) 32 bits Ordem dos dados (identifica o primeiro byte do segmento)
Número de Confirmação (ACK) 32 bits Próximo byte esperado (válido apenas se flag ACK=1)
Flags 6 bits Controle de conexão (SYN, ACK, FIN, RST, PSH, URG)
Janela (Window Size) 16 bits Quantos bytes podem ser enviados sem confirmação (controle de fluxo)
Checksum 16 bits Verificação de erros no cabeçalho e dados
Urgent Pointer 16 bits Indica dados urgentes (se URG=1)

flags do TCP

Flag Valor Função Exemplo de Uso
SYN 1 bit Synchronize - Inicia uma conexão Primeiro passo do 3-way handshake
ACK 1 bit Acknowledge - Confirma recebimento Resposta a pacotes recebidos
FIN 1 bit Finish - Encerra conexão normalmente Parte do 4-way handshake de encerramento
RST 1 bit Reset - Aborta conexão abruptamente Usado em erros ou ataques
PSH 1 bit Push - Envia dados imediatamente Evita bufferização (ex: teclado em tempo real)
URG 1 bit Urgent - Dados prioritários Pouco usado hoje (substituído por PSH)

Responsavel por estabelecer, gerenciar e encerrar sessões de comunicação entre dispositivos diferentes

Principais funções da camada de sessão

Função Explicação
Estabelecimento de sessão Cria um canal lógico entre duas aplicações
Gerenciamento de sessão Mantém e sincroniza a comunicação para garantir continuidade e eficiência
Encerramento de sessão Finaliza a comunicação de forma ordenada, liberando recursos
Sincronização Insere pontos de verificação (checkpoints) para retomar a sessão após falhas
Controle de diálogo Define se a comunicação será full-duplex ou half-duplex

full duplex/half duplex

full duplex: comunicação em dois sentidos simultaneamente, como uma chamada de video

half duplex: comunicação em um sentido por vez, como um Walkie-talkie

Exemplo da sessão em jogo

Função da Sessão Analogia no Jogo
Estabelecimento de sessão Criar a sala da partida: todos os jogadores conectam e entram na mesma sessão
Gerenciamento de sessão Acompanhar o tempo, sincronia entre ações, manter todos na mesma fase ou rodada
Encerramento de sessão Finalizar a partida e desconectar os jogadores corretamente
Sincronização Salvar progresso, pontuação, estatísticas — útil para recuperar se a conexão cair
Controle de diálogo Determinar se os jogadores podem falar ao mesmo tempo (chat de voz full-duplex) ou só por turnos (half-duplex)

Camada 6 - Apresentação

Apresenta os dados em um modelo geral para a camada de aplicação.

Tipo transforma portugues em ingles que é global, para transformar em mandarim.

Função Explicação
Tradução de dados Converte formatos diferentes (ex: ASCII ⇄ Unicode, JPEG ⇄ BMP, HTML ⇄ TXT)
Criptografia/Segurança Protege os dados através de criptografia e descriptografia
Compressão Reduz o tamanho dos dados para agilizar transmissão (ex: ZIP, MP3)
Formatação Padroniza os dados para o app entender (ex: PDFs, imagens, vídeos, textos)

Camada 7 - Aplicação

Ela permite que softwares realizem tarefas online, traduzindo ações do usuário em comandos e protocolos da rede.

Ela não é o navegador ou o app em si, mas sim a interface que conecta esses programas na rede.

Função da camada de aplicação

Função Explicação
Interface com o usuário Permite que softwares se comuniquem com a rede (ex: WhatsApp, navegador, e-mail)
Suporte a serviços de rede Fornece serviços como transferência de arquivos, envio de mensagens, etc
Identificação de protocolos Usa protocolos específicos conforme a função do app
Gerenciamento de conexão Alguns protocolos gerenciam sessões e conexões para garantir o funcionamento
Tratamento de erros Pode lidar com erros de sintaxe, protocolo e resposta inválida da rede

erros na camada de aplicação

A camada de aplicação tem liberdade para decidir o que fazer com erros — pode avisar, continuar, parar ou tentar de novo. Ela não corrige erros de rede, mas sabe como lidar com a resposta que veio falhada.

Protocolo Função/Aplicação

DNS Tradução de nomes de domínio para IP
Telnet/SSH Acesso remoto a servidores
MQTT/CoAP Comunicação com dispositivos IoT
NTP Sincronização de horário

GATEWAY CAMADA 3 ACIMA

Se os dispositivos podem se comunicar diretamente com IPs e protocolos iguais → Bridge. Se precisam de tradução ou encaminhamento entre redes diferentes → Gateway.

Se o gateway está roteando pacotes IP entre redes → atua na Camada 3 (Rede). Se está traduzindo protocolos de aplicação (como HTTP ↔ FTP) → atua na Camada 7 (Aplicação). Se está convertendo formatos de transporte (como TCP ↔ UDP) → pode envolver a Camada 4 (Transporte).
