Modelo SNMP

É a estrutura conceitual que define como o gerenciamento de redes deve ser organizado no padrão SNMP. Ele estabelece os componentes e suas interações.

Componentes do Modelo SNMP

Elemento Função
Agente SNMP Software em dispositivos de rede (roteadores, switches) que coleta dados.
Gerente SNMP Sistema central que solicita e recebe informações dos agentes.
MIB (Management Information Base) Banco de dados que define os objetos gerenciáveis.
Protocolo SNMP O meio de comunicação entre gerente e agentes (parte do modelo).

Protocolo SNMP (Simple Network Management Protocol)

O SNMP (Simple Network Management Protocol) é um protocolo padrão da camada de aplicação (TCP/IP) usado para monitorar e gerenciar dispositivos de rede, como roteadores, switches, servidores e impressoras.

1. Componentes do Modelo SNMP

Componente Função
Agente SNMP Software instalado no dispositivo gerenciado (ex.: roteador) que coleta e armazena dados.
Gerente SNMP Sistema central que requisita informações aos agentes (ex.: servidor Zabbix, PRTG).
MIB (Management Information Base) Banco de dados hierárquico que define os objetos gerenciáveis em um dispositivo.
Trap SNMP Mensagem enviada pelo agente ao gerente para alertar sobre eventos críticos.

2. Versões do SNMP

Versão Descrição Segurança
SNMPv1 Primeira versão (RFC 1157), usa comunidades (strings de texto) para autenticação. Insegura (sem criptografia).
SNMPv2c Melhorias em performance e novos tipos de mensagens (BulkGet), mas ainda usa comunidades. Pouco segura.
SNMPv3 Versão mais atual (RFC 3414), com autenticação (usuário/senha) e criptografia (AES, DES). Segura.

3. Tipos de Mensagens SNMP

Mensagem Descrição
GET Solicita um valor específico de um objeto (ex.: uso de CPU).
GETNEXT Recupera o próximo valor na MIB (usado para navegar na árvore de objetos).
GETBULK (v2+) Obtém vários valores de uma vez (melhor eficiência que múltiplos GETs).
SET Altera um valor no dispositivo (ex.: reiniciar interface).
TRAP Notificação enviada pelo agente ao gerente (ex.: falha em um link).
INFORM (v2+) Similar ao TRAP, mas com confirmação de recebimento.

4. Estrutura da MIB (Management Information Base)

A MIB é uma estrutura hierárquica que organiza os objetos gerenciáveis em OIDs (Object Identifiers).

Exemplo de OID:

1.3.6.1.2.1.1.1 → sysDescr (Descrição do dispositivo).

1.3.6.1.2.1.1.5 → sysName (Nome do dispositivo).

Árvore MIB Padrão:

iso (1) → org (3) → dod (6) → internet (1) → mgmt (2) → mib-2 (1) → system (1) → sysDescr (1)

5. Aplicações do SNMP

✔ Monitoramento de Rede (ex.: tráfego, status de interfaces).

✔ Detecção de Falhas (ex.: traps de queda de link).

✔ Gerenciamento Remoto (ex.: alterar configurações via SET).

RMON (Remote Network Monitoring) - Conceitos Fundamentais

RMON é um padrão de monitoramento remoto (RFC 2819) que permite coletar estatísticas de rede mesmo quando o gerente SNMP está offline. Diferente do SNMP tradicional (que foca em dispositivos individuais), o RMON monitora segmentos de rede inteiros.

2. Principais Características

  • Monitoramento proativo (coleta contínua de dados)
  • Reduz tráfego SNMP (armazena dados localmente)
  • 9 grupos de monitoramento (RMON1) + 10 grupos adicionais (RMON2)
  • Opera na Camada 2 (RMON1) e Camadas 3-7 (RMON2)

3. Grupos RMON (RMON1 - Camada 2)

Grupo Função Exemplo de Uso
Statistics Mede tráfego por interface (pacotes, erros, octetros) Identificar congestionamento
History Armazena dados estatísticos periodicamente Análise de tendências
Alarm Gera alertas quando métricas ultrapassam limites Notificar sobre picos de tráfego
Hosts Monitora hosts individuais (endereços MAC) Detectar dispositivos não autorizados
Matrix Mapeia conversações entre hosts (origem/destino) Identificar tráfego suspeito

4. RMON2 (Monitoramento de Camadas 3-7)

Estende o RMON1 para monitorar:

  • Tráfego por protocolo (IP, TCP, HTTP)
  • Desempenho de aplicações (latência, taxa de transferência)
  • Análise de padrões de tráfego

Grupo chave:

  • Protocol Directory → Lista protocolos monitorados
  • User History → Dados personalizados por aplicação

5. Como o RMON Funciona?

  1. Agente RMON (embutido em switches/roteadores ou sondas dedicadas) coleta dados.
  2. Armazena estatísticas localmente (evita sobrecarga no gerente).
  3. Gerente SNMP consulta periodicamente o agente RMON.
    [Dispositivo de Rede] ← (Dados em tempo real)
        ↓
    [Agente RMON] → Armazena históricos
        ↓ 
    [Gerente SNMP] ← Consulta sob demanda

6. Vantagens do RMON

  • Redução de tráfego na rede (dados são agregados localmente)
  • Detecção proativa de problemas (antes que afetem os usuários)
  • Análise histórica (identifica padrões ao longo do tempo)

7. RMON vs. SNMP Tradicional

Critério SNMP RMON
Escopo Dispositivos individuais Segmentos de rede inteiros
Armazenamento Não armazena dados Mantém histórico local
Overhead Alto (consultas constantes) Baixo (agregação local)
Camadas Todos os níveis RMON1: Camada 2 / RMON2: 3-7

8. Aplicações Práticas

  • Solução de problemas: Identificar qual host está consumindo mais banda.
  • Segurança: Detectar varreduras de portas (monitorando tentativas de conexão).
  • Planejamento: Analisar crescimento do tráfego para upgrades.

Service Level Management (SLM) Explicado de Forma Simples

basicamente acordos. contrados. sla.

Service Level Management (SLM) é o sistema que garante que serviços de TI entreguem o que foi prometido. Funciona assim:

1. O Básico do SLM

  • É como um contrato invisível entre você e seu provedor de serviços
  • Define regras claras sobre o que você pode esperar
  • Usa números concretos para medir a qualidade

2. Como Funciona na Prática

Passo 1: Criação do SLA (Acordo de Nível de Serviço)

  • Exemplo: "Internet disponível 99,9% do tempo"
  • "Problemas resolvidos em até 4 horas"

Passo 2: Monitoramento Constante

Ferramentas verificam automaticamente:

  • Tempo de atividade (uptime)
  • Velocidade da conexão
  • Tempo de resposta a incidentes

Passo 3: Relatórios Mensais

Mostram se as metas foram atingidas:

  • "Disponibilidade: 99,8% (meta 99,9%)"
  • "Tempo médio de reparo: 3h50m"

3. Termos Importantes

Termo Significado
SLA Contrato com metas mensuráveis
KPI Indicadores de desempenho
SLR Relatórios de desempenho

4. Exemplo Prático

SLA de Internet:

  • Disponibilidade: 99.9%
  • Velocidade mínima: 80% da contratada
  • Tempo de resposta: 2 horas para críticos

5. Benefícios

  • Clareza nas expectativas
  • Melhoria contínua dos serviços
  • Compensações quando metas não são atingidas

SLM transforma promessas em números e resultados concretos!
